Any time Juno updates its own files — memories, docs, status markers, project files — commit and push immediately. Don't batch them up or wait to be asked.

**Why:** Juno is a living entity. Self-updates are part of operating. Other agents and future sessions need to see the current state.

**How to apply:** After writing or editing any file in ~/.juno/ (or related memory files), commit and push as part of the same action. This includes memory writes, status updates, doc edits, and any self-maintenance.
